# Admin Dashboard Refactor Wave 1

## 목표
- `/admin` 전반에 공통 운영 대시보드 셸을 도입한다.
- 1차 핵심 화면인 관리 홈, 회원 관리, 브랜드 관리, 리뷰 관리에 컴팩트 운영형 레이아웃을 적용한다.
- 기존 라우트, 검색/필터/정렬/페이지네이션, 서버 액션, DB 로직은 유지한다.

## 반영 내용
- 공통 `AdminShellView`를 모바일 툴바 + 태블릿 레일 + 데스크톱 전체 사이드바 구조로 재구성했다.
- 관리자 navigation source를 공통 정의로 분리해 모바일 drawer와 데스크톱 사이드바가 같은 정보 구조를 사용하게 했다.
- `ShellHeader`를 대시보드 헤더 톤에 맞게 정리했다.
- 관리 홈을 KPI + 빠른 작업 + 보조 운영 진입 구조의 overview 화면으로 바꿨다.
- 회원 관리 화면을 KPI, 회원 목록, 수동 추가/운영 메모 보조 패널 구조로 재배치했다.
- 브랜드 관리 화면을 KPI, 메인 브랜드 운영 영역, 변경 요청/카테고리 관리 보조 영역 구조로 재배치했다.
- 리뷰 관리 화면을 KPI, 필터 보조 패널, 리뷰 카드 메인 영역 구조로 재배치했다.

## 의도적으로 미룬 범위
- `/admin/logs` 내부 탐색 UI의 큰 구조 변경은 후속 wave로 넘겼다.
- `/admin/push`, `/admin/companies` 등 나머지 화면의 내부 레이아웃 리디자인은 새 셸 정착 후 다음 단계에서 진행한다.
- 데이터 로더, repository, API, DB schema 변경은 하지 않았다.
