# Admin Dashboard Refactor Wave 7

## 목표
- 상세/QA 성격의 `/admin` 화면도 새 대시보드 셸 리듬에 맞춰 정리한다.
- 편집 메인 영역과 감사/미리보기 보조 영역을 분리해, 긴 세로 적층 구조를 완화한다.

## 반영 내용
- `partners/[partnerId]`에서 브랜드 수정 폼과 변경 이력을 좌우 분리해 메인 편집 영역과 감사 보조 영역의 역할을 나눴다.
- `members/mock`에 `ShellHeader`와 요약 지표를 추가해, 단순 QA 샌드박스가 아니라 관리자 내부 미리보기 도구처럼 보이게 정리했다.

## 의도적으로 미룬 범위
- `style-guide`는 실제 운영 화면이 아니라 내부 디자인 문서 성격이 강해 대시보드 구조로 다시 강제하지 않았다.
- `promotions`는 리다이렉트 경로라 별도 UI 재구성이 필요하지 않다.

## 검증
- `npx eslint` 대상 파일 통과
- `npm run build` 통과
