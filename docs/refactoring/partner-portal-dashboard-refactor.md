# Partner Portal Dashboard Refactor

## 목표
- `/partner` 하위 화면을 단순 full-width 화면이 아니라 태블릿 이상에서 운영 대시보드처럼 보이게 정리한다.
- 협력사 담당자가 브랜드 현황, 수정 요청, 알림, 지원, 계정 흐름을 좌측 내비게이션과 상단 유틸 바가 있는 업무 화면에서 처리하도록 한다.
- 화면을 끝까지 늘리는 대신 `wide` 컨테이너와 패널/보조 영역을 사용해 토스형 대시보드의 밀도와 안정감을 맞춘다.

## 반영 내용
- 로그인된 `/partner` 업무 화면에 `md` 이상 좌측 내비 레일, `xl` 이상 확장 사이드바, 상단 유틸 바를 추가했다.
- 모바일과 인증/초기 설정 화면은 기존 상단 바 흐름을 유지해 진입 장벽을 낮췄다.
- 협력사 홈은 선택 협력사 요약을 좌측 보조 패널로, 브랜드 목록을 메인 작업 영역으로 분리했다.
- 브랜드 상세 화면은 요약 카드와 이미지 영역의 비율을 넓은 화면에 맞게 조정하되 `wide` 폭 안에서 안정적으로 보이게 했다.
- 알림센터는 필터를 태블릿/데스크톱에서 보조 패널로 분리하고, 알림 목록을 메인 작업 영역으로 정리했다.
- 기술 지원 화면은 안내 패널과 문의 템플릿을 대시보드형 배치로 정리했다.
- 로그인, 비밀번호 재설정, 비밀번호 변경, 초기 설정 화면도 태블릿 이상에서 설명 영역과 입력 영역이 분리되도록 조정했다.
- `/partner` 로딩 스켈레톤도 실제 화면과 같은 대시보드형 배치 기준으로 맞췄다.

## 제외 범위
- 데이터 조회, 인증, 서버 액션, Supabase 쿼리는 변경하지 않았다.
- 공개 `/partners/[id]` 상세 화면은 이번 범위에서 제외했다.

## 검증
- 변경 파일 대상 `eslint` 통과
- `npm run build` 통과
