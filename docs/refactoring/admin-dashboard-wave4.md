# Admin Dashboard Refactor Wave 4

## 목표
- 넓어진 `/admin` 셸 폭을 핵심 운영 화면 내부 레이아웃에서도 실제로 활용한다.
- 메인 작업 영역과 보조 패널의 비율을 다시 조정해, 태블릿 이상에서 목록/결과 영역이 더 넓게 보이게 한다.
- 검색, 필터, 정렬, 페이지네이션, 상세 액션 동작은 그대로 유지한다.

## 반영 내용
- 회원 관리 화면의 메인 목록과 우측 운영 패널 비율을 메인 우선 구조로 재조정했다.
- 브랜드 관리 화면의 브랜드 운영 영역과 우측 변경 요청/카테고리 패널 비율을 메인 우선 구조로 재조정했다.
- 리뷰 관리 화면의 필터 패널 비율을 줄이고, 리뷰 카드 메인 영역이 더 넓게 보이도록 조정했다.
- 로그 탐색기의 2단 레이아웃 적용 시점을 `2xl`에서 `xl`로 당기고, 필터 패널 폭을 줄여 결과 영역 확보를 우선했다.

## 의도적으로 미룬 범위
- 각 화면 내부 카드의 정보 밀도나 리스트 item 구조 자체는 이번 wave에서 다시 설계하지 않았다.
- `/admin/push`, `/admin/companies`, `/admin/notifications`의 내부 레이아웃 정리는 후속 wave로 넘긴다.
- 로그 상세를 우측 고정 split-view로 바꾸는 interaction 수준 개편은 이번 wave에 포함하지 않았다.
