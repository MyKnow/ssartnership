# Admin Dashboard Refactor Summary

## 목표
- `/admin` 하위 화면을 공통 대시보드 셸과 운영형 레이아웃으로 정리한다.
- 모바일부터 데스크톱까지 정보 밀도를 높이고, 각 운영 화면의 역할을 분리한다.

## 진행 요약

### 1. 셸과 기본 레이아웃 정리
- `AdminShellView`를 기준으로 `/admin` 전반에 공통 셸을 적용했다.
- 태블릿 이상에서는 좌측 네비게이션과 넓은 본문 폭을 사용하도록 바꿨다.
- 공통 컨테이너 폭과 페이지 여백을 줄여 운영 화면이 가운데에 갇히지 않게 했다.

### 2. 핵심 운영 화면 재구성
- 관리 홈은 운영 요약 대시보드로 바꿨다.
- 회원 관리, 리뷰 관리, 로그 조회, 브랜드 관리, 협력사 관리, 알림 전송/센터를 대시보드형 구조로 재배치했다.
- `style-guide`는 실제 운영 화면이 아니므로 별도 대시보드 재구성 대상에서 제외했다.

### 3. 회원과 로그의 시계열 차트 정리
- 회원 유입 추이와 로그 활동량을 공통 시계열 차트 컴포넌트로 통합했다.
- 차트 높이, 여백, 축선, hover/click 인터랙션, 버블 위치, overflow 문제를 연쇄적으로 정리했다.
- 시계열 구간 선택, 지점 버블, hover 안정성, 모바일 높이 보정까지 같은 흐름으로 다듬었다.

### 4. 파트너와 브랜드 관리 재편
- 브랜드 관리와 카테고리 관리를 서로 다른 탭으로 분리했다.
- 승인 대기 요청은 브랜드 관리 탭 안에 넣어 같은 맥락에서 처리하도록 바꿨다.
- 브랜드 관리에서 페이지 폭을 밀던 고정 폭 요소와 필터 폭을 줄였다.

### 5. 상세/QA 화면 정리
- 협력사 상세, 브랜드 상세, 회원 미리보기 같은 보조 화면도 새 셸 리듬에 맞춰 정리했다.
- 상세 편집 영역과 감사/미리보기 영역을 분리해 긴 세로 적층을 줄였다.

## 통합된 버그/개선 묶음
- 차트 여백 과다, 버블 비표시, 버블 위치 어긋남, hover 잔존 버그는 하나의 시계열 상호작용 안정화 항목으로 묶었다.
- 브랜드 관리 폭 초과, 필터 최소폭, 탭 구조 분리는 하나의 파트너 운영 레이아웃 항목으로 묶었다.
- 승인 대기 요청 위치 변경과 카테고리 분리는 하나의 운영 흐름 항목으로 묶었다.

## 검증
- 변경 파일 대상 `eslint` 통과
- 필요한 범위에서 `npm run build` 통과

## 현재 상태
- 관리자 화면은 대시보드 셸과 운영형 레이아웃 기준으로 재구성된 상태다.
- 세부 수정은 앞으로도 같은 문서에 누적하지 않고, 이 요약 문서만 갱신한다.
