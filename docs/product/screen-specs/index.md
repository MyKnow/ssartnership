## 화면 계약 필드

모든 canonical 화면 계약은 다음을 빠짐없이 정의한다.

1. 사용자 목표와 정보 우선순위
2. 하나의 주 액션과 필요한 보조 액션
3. 진입점, 정상 이탈, 복귀 맥락
4. 권한과 데이터 경계
5. 기본·빈 상태·로딩·오류·권한·도메인 특수 상태
6. 모바일·태블릿·데스크톱 반응형 규칙
7. 분석 이벤트와 성공 신호
8. 구현을 판정할 수 있는 수용 기준

## routeKind

| 값 | 의미 | 화면 계약 |
| --- | --- | --- |
| `canonical` | 메뉴·링크·검색이 가리키는 유일한 제품 목적지 | 필수 |
| `conditional` | 토큰, 권한, 다중 회사 선택처럼 특정 상태에서만 렌더되는 흐름 화면 | 흐름 계약으로 관리 |
| `compat-redirect` | 이전 북마크를 canonical 경로로 보내는 호환 URL | IA에 목적지만 기록 |
| `mock-only` | 개발·합성 데이터 확인용이며 제품 내비게이션에서 숨기는 화면 | 제품 화면 계약 없음 |

## 공통 완료 조건

- 화면에는 의미상 `h1`과 primary CTA가 각각 하나만 존재한다.
- 첫 화면에서 목표, 핵심 정보, 다음 행동을 식별할 수 있다.
- 독립 의미가 없는 Card 중첩 대신 `Surface level="inset"`을 쓴다.
- 필수 상태가 실제 View Story와 360/820/1366 viewport로 연결되기 전에는 Storybook `complete`로 보지 않는다.
- 320/360/390px에서 수평 스크롤이 없고 한국어 제목·버튼이 의미를 잃게 잘리지 않는다.
- FE/BE 입력 검증은 같은 schema 또는 규칙 모듈을 사용하며 첫 오류 필드 focus를 제공한다.
