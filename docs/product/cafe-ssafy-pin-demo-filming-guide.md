# 카페 싸피 PIN 혜택 촬영 가이드

카페 싸피 제휴처의 PIN 기반 혜택 적용 방법을 SSAFY 구성원과 제휴처 직원에게 보여 주기 위한 촬영용 mock 시나리오다.

## 촬영 전 준비

1. 로컬 앱을 실행한다.

   ```bash
   npm run dev
   ```

2. 아래 주소를 브라우저에서 연다.

   ```text
   http://localhost:3000/auth/mock?returnTo=%2Fpartners%2Fcafe-ssafy-001
   ```

3. 첫 방문 시 동의 화면에서 `전체 동의하기`를 선택하고 `계속하기`를 누른다. 이 mock 경로는 동의 완료 후 카페 싸피 상세 화면으로 이동한다.

## 촬영 순서

1. 카페 싸피 상세 화면에서 대표 이미지와 추가 이미지(매장 외관, 좌석, 음료·디저트, 로고)를 보여 준다.
2. 혜택에서 `아메리카노·콜드브루 1,000원 할인`을 선택하고 수량 `1회`를 선택한다.
3. `혜택 적용하기`를 눌러 혜택 인증 화면으로 이동한다.
4. 인증 카드에 표시되는 다음 정보를 보여 준다.

   - 이름: `정민호`
   - 기수: `SSAFY 15기`
   - 캠퍼스: `서울 캠퍼스`

5. 제휴처 직원이 인증 카드를 확인한 뒤 매장 전용 PIN `0000`을 입력한다.
6. `인증 카드와 혜택 확인`을 누르고 성공 메시지를 보여 준다.
7. 실제 적용 시에는 직원이 결제 금액에서 선택한 혜택을 적용한 뒤 `혜택 이용 확인 완료`를 누르는 흐름으로 설명한다.

## 촬영용 데이터 안내

- 이름·기수·캠퍼스는 촬영용 mock 계정에 고정되어 있다.
- `0000` PIN은 촬영용 mock 데이터 전용이며 실제 운영 PIN으로 사용하면 안 된다.
- 인증 카드 사진은 `MOCK_MEMBER_PROFILE_IMAGE_URL`로 내부 경로를 지정할 수 있다. 예: `MOCK_MEMBER_PROFILE_IMAGE_URL=/mock/members/jung-minho.jpg`
- 실제 사진 파일을 준비한 뒤 해당 경로에 넣고 `.env`의 값을 설정하면 인증 카드에 표시된다. 사진이 없으면 안전한 기본 아바타가 표시된다.

## 촬영 후 확인

촬영이 끝나면 배포 환경에서 mock 변수가 활성화되지 않았는지 확인한다. 특히 `NEXT_PUBLIC_DATA_SOURCE=mock`과 `MOCK_MEMBER_AUTH=1`은 로컬 촬영 환경에서만 사용한다.
