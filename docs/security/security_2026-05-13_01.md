## 긍정적 확인 사항

- admin, user, partner session은 signed httpOnly cookie를 사용하며 production에서 `sameSite: "lax"`와 `secure`를 적용한다.
- admin login에는 account/IP throttling과 timing-safe password verification이 있다.
- 주요 민감 테이블은 schema snapshot 기준 RLS가 활성화되어 있고, broad `anon`/`authenticated` grant가 revoke되어 있다.
- review/favorite API는 signed member session을 요구하며 필요한 곳에서 ownership을 강제한다.
- suggestion API는 payload를 검증하고 HTML을 escape하며 IP 기반 rate limiting을 적용한다.
- image proxy는 private/reserved IP range를 차단하고, non-image content type을 거부하며, response size cap과 request timeout을 사용한다.
- cron endpoint는 admin session 또는 `CRON_SECRET`을 요구한다.
